# Ergo

モジュールを各ブランチで管理する。
各モジュールは独立性があるため、フォーマットとルール以外mainには何も作成しない。

## モジュールサポートリスト

| モジュール名 | 概要 | 対応 OS | 最終更新日 | ブランチ |
|-------------|------|---------|-----------|---------|
| [`ergo_input`](doc/modules/input/README.md) | マウス・キーボード・ゲームパッド・USB HID 統一入力システム | Windows / Linux / macOS | 2026-03-17 | `module/input` |

詳細は [module_list.md](module_list.md) を参照。
AI/自動処理用: [module_list.yaml](module_list.yaml)

## ブランチ運用ルール

- 各モジュールは `module/<モジュール名>` ブランチで開発する
- `main` にはドキュメントとルール定義のみ配置する
- `release` ブランチは自動生成。全モジュール情報を集積し、ドキュメント・モジュールリストを更新する
