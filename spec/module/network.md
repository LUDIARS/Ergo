# Network モジュール定義

## 概要
BSD Socket（Linux/macOS）およびWinSocket（Windows）を使用した低レイヤネットワーク通信モジュール。
HTTP リクエスト（WebRequest相当）とリアルタイム双方向通信（StreamChannel）を提供する。
Keep-Alive による接続プーリングで接続の再利用を可能にし、オーバーヘッドを削減する。

## カテゴリ
システム

## 所属ドメイン
ネットワーク / 通信

## 必要なデータ
- 接続先ホスト名・IPアドレス・ポート番号
- HTTP リクエストヘッダ・ボディ
- Keep-Alive タイムアウト値・最大接続数
- ストリームチャネル用プロトコルフレーム定義

## 依存
- なし（OS ソケット API のみ使用）

## 変数
- ソケットディスクリプタ（プラットフォーム抽象化済み）
- 接続プール（ホスト:ポート → 接続リスト）
- Keep-Alive タイムアウト（秒）
- 最大プール接続数
- ストリームチャネル状態（接続中/切断/エラー）
- 受信バッファサイズ

## 作業

### 入力
- 接続先アドレス（ホスト名またはIP + ポート）
- HTTP メソッド・パス・ヘッダ・ボディ
- ストリーム送信データ（バイナリフレーム）
- Keep-Alive 設定パラメータ

### 出力
- HTTP レスポンス（ステータスコード・ヘッダ・ボディ）
- ストリーム受信データ（バイナリフレーム）
- 接続状態イベント（接続/切断/エラー）

### タスク
- ソケット抽象化（BSD Socket / WinSocket の統一インタフェース）
- DNS 解決（ホスト名→IPアドレス変換）
- TCP 接続確立・切断
- Keep-Alive 接続プール管理（接続の再利用・タイムアウト回収・上限管理）
- HTTP リクエスト送信（GET/POST/PUT/DELETE、ヘッダ・ボディ構築・パース）
- HTTP レスポンス受信・パース（ステータス・ヘッダ・ボディ・チャンク転送対応）
- ストリームチャネル確立（TCP上の双方向リアルタイム通信）
- ストリームフレーム送受信（長さプレフィックス付きバイナリフレーム）
- ストリームイベント通知（接続・切断・受信・エラーのコールバック）
- 接続プールの定期クリーンアップ（タイムアウト接続の回収）


# テスト
- ソケットの生成・クローズが正しく動作すること
- DNS解決でホスト名からIPアドレスが取得されること
- TCP接続が確立・切断されること
- Keep-Alive接続がプールに保持され再利用されること
- プール上限到達時に最古の接続が回収されること
- タイムアウトした接続がクリーンアップされること
- HTTPリクエストが正しいフォーマットで送信されること
- HTTPレスポンスのステータス・ヘッダ・ボディが正しくパースされること
- チャンク転送エンコーディングが正しくデコードされること
- ストリームチャネルが双方向通信可能であること
- ストリームフレームの送受信でデータが欠損しないこと
- 接続・切断・エラーイベントがコールバックで通知されること
- 複数の同時接続が正しく管理されること
- プラットフォーム間でAPIが統一されていること
