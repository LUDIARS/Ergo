# Compose モジュール定義

## 概要
Arsコンパイラからの指令を受け、アプリケーション層のTypeScriptコード・Ergoモジュール・Pictor描画モジュールを結合するコンポーザーシステム。
TypeScriptで記述されたゲームロジックをC++に変換し、アクター単位のビルドとシーン単位のプレイ確認を提供する。
Web（WebGL+Wasm+TS/JS）とApp（exe+QUIC IPC）の両プラットフォームに対応する。

## カテゴリ
システム

## 所属ドメイン
ビルド / コンパイル / モジュール結合

## 必要なデータ
- TypeScript型定義ファイル（.d.ts）
- アクター定義（TSソースコード）
- Ergoモジュール依存リスト
- Pictorインタフェース定義（ObjectDescriptor等）
- ビルドターゲット情報（Web/App）
- QUICコマンド定義（予約済みシステムコマンドID）

## 依存
- Pictor（描画モジュール）
- Ergo各モジュール（input等）
- Arsランタイム（コンパイル指令発行元）

## 変数
- ビルドモード（Web / App）
- アクター結合マップ（アクター名 → 依存モジュール・描画・ロジック）
- ホットリロードフラグ
- 投機的コンパイル閾値（アイドル時間、更新頻度）
- CRC/バスト数（ファイル差分検出用）
- QUICコマンドレジストリ（予約済みID + カスタムID）

## 作業

### 入力
- Arsからのコンパイル指令（アクター/シーン単位）
- TypeScriptソースファイル
- Ergoモジュールヘッダ/ライブラリ
- Pictorインタフェース

### 出力
- 結合済みC++コード（アクター単位）
- シーンビルド成果物（Web: wasm+js / App: exe連携コマンド）
- QUICコマンドストリーム（App版）

### タスク
- TypeScript→C++トランスパイル（型定義ファイルに基づく変換）
- アクター結合（ロジック・モジュール・描画の接続コード生成）
- Webビルド（WebGL+Wasm構成、TS/JSロジック実行）
- Appビルド（exe+QUICサブモジュール連携）
- ホットリロード管理（ファイル監視、差分検出、再結合）
- 投機的C++変換（アイドル時、低頻度変更ファイルの事前コンパイル、CRC/バスト数による差分チェック）
- QUICコマンド管理（シーン再生/停止/テスト/プロファイル、予約済み+カスタムコマンド）
- シーンプレイ確認（シーン単位ビルド、Web/Appプラットフォーム切替）


# テスト
- TypeScriptコードがC++に正しく変換されること
- アクターに対してロジック・モジュール・描画が正しく結合されること
- Webビルドでwasm+jsが正しく生成されること
- Appビルドでexeとのquic通信が確立されること
- ホットリロードで変更されたロジックが即座に反映されること
- 投機的コンパイルがアイドル時に実行され、CRCで差分が検出されること
- QUICコマンドの予約済みIDが衝突しないこと
- カスタムコマンドが正しく登録・発行されること
- シーン単位のビルドでアクターが正しく配置されること
- Web/Appで異なるモジュールが選択されビルドされること
