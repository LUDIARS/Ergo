# イベントシーケンサーモジュール定義

## 概要
ゲーム中にイベントを発行し、キューイングしつつ順次処理するシーケンサーシステム。
イベントは優先度付きキューで管理され、フレーム単位で順次ディスパッチされる。

## カテゴリ
システム

## 所属ドメイン
イベント / シーケンス制御

## 必要なデータ
- イベントID（型識別用）
- イベントペイロード（任意データ）
- 優先度（処理順序の制御用）

## 依存
- なし

## 変数
- イベントキュー（優先度付き）
- リスナーマップ（イベントID → コールバックリスト）
- フレームあたり最大処理数
- キュー最大サイズ
- 処理中フラグ（再帰防止）

## 作業

### 入力
- イベント（ID + ペイロード + 優先度）
- リスナー登録（イベントID + コールバック）

### 出力
- リスナーへのイベントディスパッチ

### タスク
- イベントのキューへの追加（優先度付き）
- キューからのイベント取り出しとディスパッチ
- リスナーの登録・解除
- フレーム単位の処理数制限
- 遅延イベントの発行（指定フレーム後に発行）
- キューのクリア
- 処理中の再帰的イベント発行の安全なハンドリング


# テスト
- イベントがキューに追加され順次処理されること
- 優先度の高いイベントが先に処理されること
- リスナーが正しく登録・解除されること
- フレームあたり最大処理数を超えた場合に次フレームに持ち越されること
- 遅延イベントが指定フレーム後に発行されること
- キューが空の場合に何も処理されないこと
- 処理中にイベントを発行しても安全に動作すること
- リスナーが未登録のイベントが発行されても例外が起きないこと
- キューのクリアで全イベントが除去されること
- キュー最大サイズ超過時にイベントが破棄されること
