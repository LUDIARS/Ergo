# Sound モジュール定義

## 位置付け (Core Pillar)

**サウンドは Ergo のコア柱であり、`rendering` と並ぶ主要実装である**。
外部ミドルウェアの薄いラッパや、ゲームの一機能として従属させる扱いは
しない。自前デコーダ・自前ミキサ・自前 DSP・自前スケジューラを持つ
**ファーストクラス** 実装として発展させる:

- 音響表現 (DSP / ノーマライズ / ピッチ / エンベロープ / クォンタイズ) を
  フレームワーク内で完結させ、外部 SDK のロードマップに依存しない。
- BPM 同期 / グリッドスナップ / 波形操作はサウンドゲームや音楽ゲーム的
  表現の基盤として、`rendering` (pictor) のマテリアル / シェーダと同等に
  継続投資する領域。
- FMOD 等のプロ向け SDK はゲーム SE/BGM の「本番配信ライン」(`ergo_audio`)
  に限定。`ergo_sound` は**それと独立した** 音響エンジンとして、ミドル
  ウェアが存在しない環境でも完全なサウンド体験を提供する。

`ergo_audio` (ゲーム SE ファサード) との役割分担:

| 観点 | `ergo_sound` | `ergo_audio` |
|------|--------------|--------------|
| 位置付け | コア柱 / 自前実装 | ゲーム FX ファサード / SDK 抽象 |
| 用途 | 音楽ゲーム、DSP、DAW 連携、波形表現 | ゲーム SE / BGM ワンショット |
| 依存 | なし (Threads のみ) | FMOD Core (無ければ Dummy) |
| 発展方向 | DSP / スケジューラ / 波形合成 | バックエンド差替え (Wwise 等) |

## 概要
サウンドファイル（WAV/OGG等）の読み込み・デコード、ファイルストリーミング再生、高優先度スレッドによるリアルタイムオーディオ出力、波形操作、クォンタイズ機能を提供するオーディオシステム。

## カテゴリ
システム (コア柱)

## 所属ドメイン
オーディオ / サウンド (プラットフォーム非依存のコア音響層)

## 必要なデータ
- サウンドファイル（WAV, OGG）
- オーディオ出力デバイス情報
- BPM・拍子情報（クォンタイズ用）

## 依存
- なし（外部ライブラリ不要、自前デコーダ）
  — これは **設計上の強い制約**。FMOD / miniaudio / OpenAL 等のプロプライ
  エタリ依存を入れない。新機能は自前実装で拡張する。

## 変数
- サンプルレート
- チャンネル数
- ビット深度 / サンプルフォーマット
- デコードバッファサイズ
- ストリームバッファサイズ
- オーディオスレッド優先度
- ボイス数上限
- BPM
- クォンタイズグリッドサイズ

## 作業

### 入力
- サウンドファイルパス
- PCMデータバッファ
- 波形操作パラメータ（ゲイン、フェード、ピッチ等）
- BPM・グリッド設定

### 出力
- デコード済みPCMデータ
- ストリーミング再生状態
- ミキシング済みオーディオ出力
- 波形操作結果
- クォンタイズ済みサンプル位置

### タスク
- WAVファイルのヘッダ解析とPCMデコード
- OGGファイルのVorbisデコード
- ファイルストリーミング（部分読み込みによる逐次デコード再生）
- 高優先度オーディオスレッドの生成とリアルタイムミキシング
- ボイス管理（再生・停止・一時停止・再開）
- 波形の振幅操作（ゲイン、ノーマライズ）
- フェードイン・フェードアウト
- ピッチシフト
- 波形ミキシング
- エンベロープ適用
- BPMベースのクォンタイズ（サンプル位置→グリッド位置のスナップ）
- ビート・サンプル間の相互変換


# テスト
- WAVファイルのヘッダが正しく解析されること
- WAVファイルのPCMデータが正しくデコードされること
- OGGファイルが正しくデコードされること
- 不正なファイルに対してエラーが返されること
- ストリームが部分的にデータを読み込めること
- ストリーム終端が正しく検出されること
- ストリームのシークが正しく動作すること
- オーディオエンジンが初期化・終了できること
- ボイスの再生・停止・一時停止・再開が正しく動作すること
- 複数ボイスがミキシングされること
- ゲイン適用で振幅が正しく変化すること
- フェードイン・フェードアウトが正しく適用されること
- ピッチシフトが正しく動作すること
- ノーマライズで最大振幅が1.0に正規化されること
- 2つの波形が正しくミキシングされること
- クォンタイズでサンプル位置が最寄りのグリッドにスナップされること
- BPMとサンプルレートに基づくビート・サンプル変換が正しいこと
