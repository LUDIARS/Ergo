# Sound モジュール定義

## 概要
WAV サウンドファイルの読み込み・デコード、ファイルストリーミング再生、
高優先度スレッドによるリアルタイムオーディオ出力、波形操作、クォンタイズ機能を
提供するオーディオシステム。

外部依存を避けるため自前デコーダのみを抱えており、現時点では WAV (PCM Int16 /
Int24 / Float32) 以外のフォーマットはサポートしない。OGG Vorbis 等の
圧縮フォーマットが必要になったら stb_vorbis を `third_party/` に vendor して
別モジュール / 別コミットで足すこと。

## カテゴリ
システム

## 所属ドメイン
オーディオ / サウンド

## 必要なデータ
- サウンドファイル（WAV: PCM Int16 / Int24 / IEEE Float32）
- オーディオ出力デバイス情報
- BPM・拍子情報（クォンタイズ用）

## 依存
- なし（外部ライブラリ不要、自前デコーダ）

## 変数
- サンプルレート
- チャンネル数
- ビット深度 / サンプルフォーマット
- デコードバッファサイズ
- ストリームバッファサイズ
- オーディオスレッド優先度
- ボイス数上限
- BPM
- クォンタイズグリッドサイズ

## 作業

### 入力
- サウンドファイルパス
- PCMデータバッファ
- 波形操作パラメータ（ゲイン、フェード、ピッチ等）
- BPM・グリッド設定

### 出力
- デコード済みPCMデータ
- ストリーミング再生状態
- ミキシング済みオーディオ出力
- 波形操作結果
- クォンタイズ済みサンプル位置

### タスク
- WAVファイルのヘッダ解析とPCMデコード（Int16 / Int24 / Float32）
- ファイルストリーミング（部分読み込みによる逐次デコード再生）
- 高優先度オーディオスレッドの生成とリアルタイムミキシング
- ボイス管理（再生・停止・一時停止・再開）
- 波形の振幅操作（ゲイン、ノーマライズ）
- フェードイン・フェードアウト
- ピッチシフト
- 波形ミキシング
- エンベロープ適用
- BPMベースのクォンタイズ（サンプル位置→グリッド位置のスナップ）
- ビート・サンプル間の相互変換


# テスト
- WAVファイルのヘッダが正しく解析されること
- WAVファイルのPCMデータが正しくデコードされること (Int16 / Int24 / Float32)
- `.ogg` 等の未対応拡張子で `IAudioDecoder::create` が nullptr を返すこと
- 不正なファイルに対してエラーが返されること
- ストリームが部分的にデータを読み込めること
- ストリーム終端が正しく検出されること
- ストリームのシークが正しく動作すること
- オーディオエンジンが初期化・終了できること
- ボイスの再生・停止・一時停止・再開が正しく動作すること
- 複数ボイスがミキシングされること
- ゲイン適用で振幅が正しく変化すること
- フェードイン・フェードアウトが正しく適用されること
- ピッチシフトが正しく動作すること
- ノーマライズで最大振幅が1.0に正規化されること
- 2つの波形が正しくミキシングされること
- クォンタイズでサンプル位置が最寄りのグリッドにスナップされること
- BPMとサンプルレートに基づくビート・サンプル変換が正しいこと
